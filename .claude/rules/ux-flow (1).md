# CaféLog — UX Flow

## 플로우 구조

서비스는 크게 세 가지 핵심 행동으로 구성된다:
**기록하기 / 탐색하기 / 모아보기**

---

## 시나리오 1 — 카페 방문 후 기록하기

사용자가 카페를 방문한 뒤, 그 경험을 기록으로 남기는 흐름.

> 앱 진입
> → 방문한 카페 선택 (검색 또는 지도에서 직접)
> → 음료 종류 지정
> → 카페 유형 태그 선택 (복수 선택)
> → 사진 추가
> → 메모 입력 (선택)
> → 공개 여부 설정
> → 기록 저장

---

## 시나리오 2 — 가고 싶은 카페 탐색하기

지도 또는 피드에서 카페를 발견하고 저장하는 흐름.

> 앱 진입
> → 지도 뷰에서 목적·테마·음료 조건으로 필터링
> → 관심 있는 카페 선택
> → 해당 카페의 방문자 기록 확인
> → 카페 저장 (북마크)

---

## 시나리오 3 — 피드에서 다른 사람의 기록 탐색하기

팔로우 중인 사용자 또는 전체 공개 기록을 탐색하는 흐름.

> 피드 진입
> → 기록 카드 탐색 (카페명·태그·사진 기반)
> → 관심 있는 기록 확인
> → 해당 카페 저장 또는 작성자 팔로우

---

## 시나리오 4 — 내 기록 모아보기

내가 방문한 카페의 기록을 시간순으로 되돌아보는 흐름.

> 마이페이지 또는 캘린더 진입
> → 월별 방문 기록 확인
> → 특정 날짜의 기록 상세 확인

---

## 탐색 구조

서비스는 다섯 개의 주요 영역으로 구성된다:
지도 탐색 / 피드 / 기록하기 / 캘린더 / 마이페이지
